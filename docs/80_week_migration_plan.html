Option A Data Model — Semi-Structured (Delinea Platform + StrongDM)

Folder-based hierarchy with template-driven secrets (78 built-in + custom). 200+ granular role-level permissions + 4-tier folder permissions (Owner/Edit/Add Secret/View). Requires multiple infrastructure components: Distributed Engine (Windows, RPC/heartbeat), RabbitMQ (TCP 5671/5672), and StrongDM gateways/relays for continuous authorization. More modular and cloud-native, but more integration work.

Option B Data Model — Cohesive Relational (CyberArk Privilege Cloud)

Safe → Account → Platform model with consistent hierarchy. 22 individual permissions per safe member (1:1 from on-prem). Same PVWA REST API surface, single vendor, single API, single console. Tighter integration with less infrastructure overhead. Less work to migrate (same vendor), but less modular.

Reusable Migration Framework

This plan is a vendor-agnostic PAM migration framework — not tied to any single source or target platform. The PamVendorAdapter interface, multi-agent orchestration, and phased lifecycle can be applied to any PAM migration: CyberArk → Delinea, BeyondTrust → CyberArk, HashiCorp Vault → any target, etc. Options A and B demonstrate two concrete target architectures. New vendor adapters plug in without changing the portal, API, or migration agents.

Keep CyberArk running in read-only mode for the audit retention period (1-7 years depending on compliance framework). Audit logs and PSM recordings do NOT transfer to Secret Server. Do not decommission until all audit obligations are met.
